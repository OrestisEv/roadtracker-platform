# Contributing to RoadTracker

## How to contribute

1. Read the [Feature Matrix](FEATURE_MATRIX.md) and [Change Process](docs/change-process.md) first
2. Determine which repo your change belongs to (platform, Android, or web)
3. Fork the relevant repository
4. Make your changes on a branch
5. Submit a pull request to the main repository
6. Only contributions merged by the maintainer become official

## Which repo?

| Your change affects... | Submit PR to |
|---|---|
| Database schema, shared contracts, docs | `roadtracker-platform` |
| Android app only | `Riders-track-map` |
| Web app only | `road-tracker-web` |
| Both apps (Tier 1 feature) | Start with `roadtracker-platform`, then PRs to both app repos |

## Contributor pool and profit sharing

RoadTracker operates on a good-faith stakeholder model.

**Who is in the pool:** every contributor who has had at least one pull request accepted and merged into any RoadTracker repository.

**If the project is ever commercialized:** the maintainer (Oresti Ev) will convene the Contributor Pool. Members will vote collectively to decide how a portion of the revenue or proceeds is distributed.

The maintainer retains final decision-making authority over whether and how to commercialize. The profit-sharing model is a good-faith commitment, not a legally binding contract. Participation does not constitute employment, partnership, or equity ownership.

**What counts as a contribution:**
- Accepted code changes (merged pull requests) to any RoadTracker repo
- Significant accepted documentation or design contributions at the maintainer's discretion

## By submitting a pull request, you confirm that:

- You wrote the code yourself or have the right to submit it
- You agree to the LICENSE terms of this project
- You grant the maintainer the rights described in the LICENSE
- You understand the good-faith nature of the profit-sharing model
